# AI Usage

AI Usage is a Windows tray dashboard for AI coding subscription usage. Keep it lightweight, focused, and practical for daily desktop use.

## Instructions

- Use simple, concise language.
- Be precise. Avoid fluff.
- Do not over-engineer; keep changes focused on usage tracking and Windows tray workflows.

## Guardrails

- Bugs should include regression tests when practical.
- Keep edits scoped to the requested behavior.
- Avoid unrelated refactors.
- Never delete files, folders, or user data unless explicitly approved or part of an agreed plan.
- For provider changes, avoid logging tokens, account secrets, or raw credential payloads.

## Research

- Prefer official docs and primary sources when current provider behavior or external APIs may have changed.
- Quote exact errors when debugging.
- Prefer 2025-2026+ sources for fast-moving provider or platform behavior.

## Error Handling

- Expected issues should be represented clearly in UI or plugin output.
- Unexpected issues should fail loudly through logs or explicit errors; do not add silent fallbacks that hide real failures.

## Before Release Or Main Push

- Ensure `README.md` lists the supported bundled providers.
- Ensure `CHANGELOG.md` includes project-local release notes.
- On provider changes, audit request/response fields against `src-tauri/src/plugin_engine/host_api.rs` redaction coverage.
- In `plugin.json`, keep `brandColor` aligned with the provider brand.
- Plugin SVG logos should use `currentColor` so icon theming works correctly.
- For visual changes, provide before/after screenshots when practical.

## Project Notes

- Tauri IPC: JS must use camelCase (`{ batchId, pluginIds }`); Tauri converts to Rust snake_case. Do not send snake_case from JS.
- Tauri release artifacts are generated by `npm.cmd run tauri -- build`.
